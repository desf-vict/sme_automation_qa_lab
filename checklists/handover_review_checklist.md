# Handover Review Checklist

## Purpose

This checklist verifies whether the customer can receive, understand, and operate the automation.

It is used after functional QA and before customer delivery.

---

## Handover Material Checks

| Check | Expected Result | Status | Notes |
|---|---|---|---|
| Setup guide available | Customer can install or prepare environment | Pending | |
| Run guide available | Customer can execute automation | Pending | |
| Input format documented | Customer knows required input | Pending | |
| Output format documented | Customer knows expected output | Pending | |
| Error messages documented | Customer knows what common errors mean | Pending | |
| Known limitations documented | Customer understands constraints | Pending | |
| Support route documented | Customer knows who to contact | Pending | |

---

## Operational Checks

| Check | Expected Result | Status | Notes |
|---|---|---|---|
| Customer can locate input folder | Input path understood | Pending | |
| Customer can locate output folder | Output path understood | Pending | |
| Customer can run the automation | Execution steps understood | Pending | |
| Customer can identify success | Success criteria visible | Pending | |
| Customer can identify failure | Failure criteria visible | Pending | |

---

## Final Handover Decision

| Decision | Meaning |
|---|---|
| Handover ready | Customer can receive the automation |
| Handover ready with notes | Minor limitations documented |
| Not ready | Missing instructions or unresolved blocker |
| Customer training needed | Delivery requires walkthrough |

---

## QA Principle

A technically working automation is not complete if the customer cannot operate it.
